你描述的这种功能全面的工具，目前市面上还没有一个现成的、能完全集成所有你列出的数据源（剪贴板历史、窗口ID/标题、虚拟桌面ID、键盘输入、主要选择区）并同时提供时间线浏览和关键词搜索的软件。

不过，确实存在一些能覆盖其中部分功能的工具，主要集中在 **Linux** 平台。

### 🔍 现有工具能做什么？

*   **`life-monitor` (Linux/Windows)**：这是一个跨平台的 Rust 活动追踪器，能记录键盘、鼠标活动和当前聚焦的窗口，数据存储在 SQLite 数据库中。但它**不记录剪贴板、虚拟桌面或主要选择区**，也没有图形界面，需要用户自己通过 SQL 查询数据。

*   **`LoggerMan` (似乎主要针对 Linux)**：一个学术研究项目，功能非常全面，能记录**键盘、鼠标、剪贴板、应用程序切换和屏幕截图**。它提供了一个**可缩放的时间线视图**来浏览应用和截图，并支持通过标签云（如窗口标题）进行浏览。但它**不记录虚拟桌面和主要选择区**，且项目看起来已不再活跃。

*   **剪贴板管理器 (Linux)**：
    *   **`Diodon`**：能与 Zeitgeist 日志框架集成，提供无限的历史记录，并同步 **`Ctrl+C`/`Ctrl+V` 剪贴板**和 **X11 主要选择区 (Primary Selection)**。
    *   **`Clipman`**：同样支持主要选择区。
    *   **`wmcliphist`**：专门为 X11 选择区（包括 PRIMARY 和 CLIPBOARD）提供历史记录。
    但这些工具都**不记录窗口、虚拟桌面或键盘输入**。

*   **`ClipGenius Pro` (Windows/macOS)**：一个本地优先的剪贴板管理器，能将剪贴板历史组织成**可搜索的工作会话**，并提供**时间线视图**。但它**不记录窗口、虚拟桌面、键盘或主要选择区**。

*   **Windows 原生功能**：Windows 系统自带的剪贴板历史（`Win+V`）仅能记录文本和图片。Windows Recall 功能虽能记录全面的 PC 活动，但通过**屏幕截图**实现，并非记录你指定的结构化数据。

*   **Android**：目前**没有找到**任何能实现类似功能的工具。这主要受限于系统对后台活动和跨应用数据访问的严格限制。

### 🛠️ 如何自己构建？

既然没有现成的，你可以考虑组合或自己开发一个。核心思路是**模块化**：

1.  **数据收集**：为每种数据编写单独的采集模块。
    *   **剪贴板 & 主要选择区**：在 Linux (X11) 上，可参考 `wmcliphist`或 `Diodon`的思路，监听 `CLIPBOARD` 和 `PRIMARY` 选择区的变化。
    *   **窗口信息**：在 Linux (X11) 上，可使用 `xdotool` 或 `Xlib` 获取当前聚焦窗口的 ID 和标题。
    *   **虚拟桌面 ID**：在 KDE 上，可通过 `qdbus` 查询 (`org.kde.KWin` / `VirtualDesktopManager`) 获取。
    *   **键盘输入**：在 Linux 上，读取 `/dev/input/` 设备或使用 `libevdev` 库。

2.  **数据存储**：将所有带时间戳的数据存入一个数据库，**SQLite** 是很好的选择。

3.  **用户界面**：开发一个界面，用于按时间倒序浏览数据库记录，并提供关键词搜索功能。

### ⚠️ 各平台实现限制

*   **Linux (X11)**：**可行性最高**。上述所有数据点都可以通过公开的接口或标准协议获取，是唯一理论上可以实现全部功能的平台。
*   **Windows**：**可实现大部分功能**。剪贴板、窗口、键盘输入都有成熟的 API。但“虚拟桌面”的 API (`IVirtualDesktopManager`) 功能有限，且**没有“主要选择区”** 的概念。
*   **macOS**：**可实现大部分功能**。有相应的 API，但权限申请严格。同样**没有“主要选择区”**，“虚拟桌面”（Spaces）的 API 也不透明。
*   **Android**：**可行性极低**。系统安全策略**严格限制**了后台活动和跨应用数据访问，几乎不可能实现。

### 💎 总结

一个完全满足你所有要求的“万能”工具并不存在。目前最接近的是 **Linux 平台**上的 **`LoggerMan`**（有时间线和多种数据）和 **`life-monitor`**（有窗口和键盘），但都不完美。

如果你确实需要这样的工具，尤其是在 Linux (Kubuntu) 上，**自行开发**可能是最现实的路径。可以先从组合 `life-monitor`（记录窗口、键盘）和一个支持主要选择区的剪贴板管理器（如 `Diodon`）开始，再通过脚本将它们的数据库关联起来，作为一个可行的起点。
